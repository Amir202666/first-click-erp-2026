# خطوات تشغيل النظام خطوة بخطوة

> **سير العمل (محلي → GitHub → سيرفر):** [`docs/WORKFLOW-LOCAL-TO-PRODUCTION.md`](docs/WORKFLOW-LOCAL-TO-PRODUCTION.md)  
> **إعداد محلي سريع:** شغّل `scripts\local-bootstrap.bat` ثم `scripts\local-dev.cmd`

## ⚠️ رفع التعديلات أونلاين (مهم)

**`git push` يحدّث GitHub فقط — لا يحدّث الموقع الحي تلقائياً** ما لم تُفعَّل GitHub Actions (أسرار السيرفر).

| الخطوة | ماذا تفعل |
|--------|-----------|
| 1 | `git push origin main` (من جهازك) |
| 2 | **نشر على السيرفر** — أحد الخيارين أدناه |
| 3 | تحقق: `scripts\verify-production.bat` أو افتح `/deploy-revision.txt` |
| 4 | في المتصفح: **Ctrl+Shift+R** أو نافذة خاصة |

### الخيار أ — من Windows (إن كان SSH يعمل)

```bat
scripts\deploy-to-server.bat
```

### الخيار ب — Hostinger VPS → Browser Terminal (الأضمن)

```bash
cd /var/www/erp && git fetch origin main && git reset --hard origin/main && bash deploy.sh
```

بعد النشر يجب أن يظهر في `https://firstclickerp.top/deploy-revision.txt` نفس hash آخر commit (مثل `50e7602`).

**نشر تلقائي (اختياري):** أضف أسرار `SERVER_HOST` و `SERVER_USER` و `SERVER_PASSWORD` في GitHub → Settings → Secrets → Actions، ثم كل push على `main` ينشر وحده.

## تشغيل إنتاج مستقر (مُوصى به لتجنب انقطاع الاتصال)

للتوقف عن الاعتماد على `php artisan serve` و `npm run dev` وتشغيل النظام على خادم محلي ثابت (Apache/Nginx) مع قاعدة بيانات كخدمة:

1. نفّذ **`build-production.cmd`** لبناء الواجهة ونسخها إلى الخادم.
2. اتبع دليل **`تشغيل-الإنتاج-Laragon-XAMPP.md`** لإعداد Laragon أو XAMPP وقاعدة البيانات (MySQL).

بعد الإعداد، يُفتح التطبيق من رابط ثابت (مثل `http://first-click.test`) دون الحاجة لفتح نوافذ أوامر.

---

## تشغيل التطوير (للاختبار السريع فقط)

عند ظهور رسالة **"فشل الاتصال بالخادم"** استخدم إحدى الطرق أدناه.

### الطريقة السريعة

1. من مجلد المشروع، شغّل الملف **`تشغيل-النظام.cmd`** (نقرة مزدوجة).
2. ستُفتح نافذتان: واحدة للخادم الخلفي وأخرى للواجهة. **لا تغلقهما**.
3. بعد ظهور "Local: http://localhost:5173" في نافذة الواجهة، افتح المتصفح على: **http://localhost:5173**.

---

## الطريقة اليدوية

### الخطوة 1: تشغيل الخادم الخلفي (Laravel)

1. افتح **موجه الأوامر** (Command Prompt) أو **Terminal**.
2. انتقل إلى مجلد الخادم:
   ```
   cd "d:\erp projects\first click\backend"
   ```
3. شغّل الخادم بالأمر:
   ```
   php artisan serve
   ```
4. انتظر حتى تظهر رسالة مثل:
   ```
   Server running on [http://127.0.0.1:8000]
   ```
5. **لا تغلق هذه النافذة** — اتركها مفتوحة طوال فترة العمل.

---

## الخطوة 2: تشغيل الواجهة الأمامية (React)

1. افتح **نافذة جديدة** من موجه الأوامر (لا تغلق نافذة الخادم).
2. انتقل إلى مجلد الواجهة:
   ```
   cd "d:\erp projects\first click\frontend"
   ```
3. شغّل الواجهة:
   ```
   npm run dev
   ```
4. انتظر حتى يظهر عنوان مثل:
   ```
   http://localhost:5173
   ```
5. **لا تغلق هذه النافذة** — اتركها مفتوحة أيضاً.

---

## الخطوة 3: التحقق من أن الخادم الخلفي يعمل

1. افتح المتصفح (Chrome أو Edge).
2. في شريط العنوان اكتب:
   ```
   http://127.0.0.1:8000/api/health
   ```
3. اضغط Enter.
4. إذا كان الخادم الخلفي يعمل بشكل صحيح ستظهر عبارة مثل: `{"ok":true}`.
5. إذا لم تظهر أو ظهر خطأ، راجع **الخطوة 1** وتأكد أن نافذة `php artisan serve` مفتوحة ولا تظهر فيها أخطاء.

---

## الخطوة 4: فتح التطبيق

1. في المتصفح اكتب في شريط العنوان:
   ```
   http://localhost:5173
   ```
2. اضغط Enter.
3. **مهم:** استخدم دائماً هذا العنوان (`http://localhost:5173`) وليس `http://127.0.0.1:8000`.

---

## الخطوة 5: تسجيل الدخول واستخدام التطبيق

1. سجّل الدخول بحسابك إذا طُلب منك ذلك.
2. اختر الفرع/الشركة إذا ظهر خيار الاختيار.
3. يمكنك الآن تعديل الفواتير أو أي عملية أخرى — يجب أن يعمل الحفظ بدون "فشل الاتصال بالخادم".

---

## ملخص سريع

| الترتيب | ماذا تفعل |
|--------|------------|
| 1 | من مجلد `backend`: تشغيل `php artisan serve` وترك النافذة مفتوحة |
| 2 | من مجلد `frontend`: تشغيل `npm run dev` وترك النافذة مفتوحة |
| 3 | التحقق من الخادم الخلفي: فتح `http://127.0.0.1:8000/api/health` في المتصفح |
| 4 | فتح التطبيق من `http://localhost:5173` واستخدامه |

## إذا استمرت المشكلة

- تأكد أن **نافذتا** الأوامر (الخادم والواجهة) مفتوحتان ولا تظهر فيهما أخطاء.
- تأكد أنك تفتح التطبيق من **http://localhost:5173** وليس من عنوان آخر.
- جرّب إعادة تشغيل الكمبيوتر ثم تنفيذ الخطوات من جديد.

## اختبار المشروع بـ TestSprite (Cursor)

> **سبب الفشل الشائع:** مفتاح API فارغ في إعدادات Cursor — بدون المفتاح لن يعمل TestSprite أبداً.

### 1) الحصول على مفتاح API

1. سجّل دخولاً أو أنشئ حساباً: https://www.testsprite.com  
2. من **Dashboard → Settings → API Keys** أنشئ مفتاحاً جديداً (**New API Key**).  
3. انسخ المفتاح (يظهر مرة واحدة فقط).

### 2) وضع المفتاح في Cursor

1. افتح الملف (بالمحرر أو Notepad):

   `C:\Users\Amir\.cursor\mcp.json`

2. استبدل السطر الفارغ:

   ```json
   "API_KEY": ""
   ```

   بـ:

   ```json
   "API_KEY": "ts_xxxxxxxxxxxxxxxx"
   ```

   (ضع مفتاحك الحقيقي مكان `ts_xxx...`)

3. **أغلق Cursor بالكامل** ثم افتحه من جديد (إعادة تشغيل إلزامية).

4. في **Cursor → Settings → Tools & MCP** تأكد أن **TestSprite** يظهر بنقطة خضراء وأن الأدوات محمّلة.

5. في **Chat → Auto-Run** اختر **Ask Everytime** أو **Run Everything** (وليس Sandbox فقط).

### 3) تشغيل التطبيق قبل الاختبار

| الخدمة | الأمر | الرابط للتحقق |
|--------|--------|----------------|
| Backend | `cd backend` ثم `php artisan serve` | http://127.0.0.1:8000/api/health |
| Frontend | `cd frontend` ثم `npm run dev` | http://localhost:5173 |

### 4) تشغيل TestSprite من الشات

اكتب في شات Cursor:

```text
Help me test this project with TestSprite.
```

سيُفتح **متصفح** لصفحة الإعداد. املأ:

| الحقل | القيمة المقترحة |
|--------|------------------|
| النوع | Frontend (ولاحقاً Backend للـ API) |
| النطاق | Codebase |
| رابط الواجهة | `http://localhost:5173` |
| رابط الـ API | `http://127.0.0.1:8000` |
| تسجيل الدخول | `admin@test.com` / `password` (إن وُجد المستخدم — انظر README) |

### 5) اختبار بديل (بدون TestSprite) — Backend

إذا لم يتوفر مفتاح TestSprite، شغّل اختبارات Laravel المحلية:

```bash
cd backend
php artisan test
```
