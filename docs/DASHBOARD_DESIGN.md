# لوحة القيادة التفاعلية (Interactive Dashboard)

## نظرة عامة

لوحة تحكم احترافية لنظام محاسبي و POS متعدد الشركات (Multi-tenant)، تجمع بين واجهة حديثة (Dark/Light) وتحليلات متقدمة مع خصوصية البيانات حسب `Company_ID` (tenant).

---

## 1. قسم المؤشرات اللحظية (The Pulse)

- **صافي الربح** و **رصيد البنك**: أكبر رقمين في الشاشة (بطاقة Hero).
- **Sparkline (تدفق السيولة)**: رسم مصغر للمبيعات اليومية خلال الفترة المختارة (Recharts AreaChart).
- البيانات تُجلب من الـ API: `pulse.net_profit`, `pulse.cash_flow_sparkline`.

---

## 2. التحليل التنبؤي (Predictive Insights)

- **توقعات مبيعات الشهر القادم**: متوسط المبيعات للأشهر السابقة (بسيط).
- **معدل استهلاك السيولة (Cash Burn Rate)**: مصروفات الشهر السابق ÷ أيام الشهر.
- الـ API يُرجع: `predictive.next_month_sales_forecast`, `predictive.cash_burn_rate_per_day`.

---

## 3. الخرائط الحرارية (Heatmaps)

- **أوقات الذروة في نقطة البيع (POS Peak Hours)**: شبكة 24 ساعة، لون أخضر يزداد مع زيادة المبيعات/عدد العمليات في كل ساعة.
- البيانات: `pos_peak_hours[]` مع `hour`, `count`, `total`.

---

## 4. نظام التنبيهات الذكي (Actionable Notifications)

- **بطاقات تنبيه** قابلة للنقر:
  - "هناك X فواتير مشتريات مستحقة اليوم" → ينتقل إلى الفواتير.
  - "صنف قارب على الانتهاء" (أصناف تحت الحد الأدنى) → ينتقل إلى تقرير المخزون.
- المصدر: `notifications.due_purchase_count`, `notifications.low_stock_count`, `notifications.due_purchase_invoices`.

---

## 5. الفلترة الديناميكية (Dynamic Filtering)

- **الفترة**: اليوم / هذا الأسبوع / هذا الشهر / السنة / تاريخ مخصص.
- **الفرع**: قائمة منسدلة (جميع الفروع أو فرع محدد).
- التغيير يحدّث استدعاء الـ API دون إعادة تحميل الصفحة (SPA):  
  `GET /dashboard?period=...&from_date=...&to_date=...&branch_id=...`

---

## 6. توزيع المصروفات (Expense Breakdown)

- **رسم دائري تفاعلي (Donut)**: توزيع المصروفات حسب الحساب (Recharts PieChart).
- ألوان هادئة (Pastel).
- الـ API يُرجع: `expense_breakdown[]` مع `account_id`, `account_name`, `amount`, `journal_entry_ids` (للربط بالقيود لاحقاً).

---

## 7. فجوة المبيعات والودائع (Gap Analysis)

- مقارنة **المبيعات المتوقعة** مع **المبالغ المودعة في البنك** خلال الفترة.
- يوضح **الفجوة** (عجز أو زيادة) لاكتشاف أي تلاعب أو عجز في الصندوق.
- الـ API: `gap_analysis.expected_sales`, `gap_analysis.bank_deposits`, `gap_analysis.gap`.

---

## 8. المواصفات التقنية

- **الرسوم البيانية**: Recharts (Bar, Area, Pie).
- **السحب والإفلات**: يمكن إضافة react-grid-layout لاحقاً لترتيب البطاقات (النسخة الحالية تستخدم CSS Grid ثابت).
- **الربط بـ Company_ID**: جميع استدعاءات الـ API تستخدم `X-Tenant-ID` و`branch_id` (اختياري) لضمان عرض بيانات الشركة/الفرع المختار فقط.

---

## 9. وضع الخصوصية (Privacy Mode)

- زر **وضع الخصوصية**: يغيّش (Blur) الأرقام المالية في اللوحة.
- يسمح بفتح الشاشة أمام آخرين دون كشف الأرباح أو السيولة أو الرصيد.

---

## 10. البحث السريع (Command Palette)

- اختصار **Ctrl+K** (أو Cmd+K) لفتح شريط بحث.
- أمثلة أوامر:
  - "عرض مبيعات أمس" → ضبط الفترة على اليوم.
  - "عرض مبيعات الشهر" → ضبط الفترة على الشهر.
  - "الذهاب إلى الفواتير" / "فتح نقطة البيع" → تنقل مباشر.

---

## 11. واجهة المستخدم (UI/UX)

- ألوان هادئة (Pastel) للرسوم لتقليل تشتت العين.
- **صافي الربح** و **رصيد البنك** هما الأبرز في الشاشة (Hero).
- دعم الوضع الداكن (Dark) والفاتح (Light) عبر ThemeContext.
