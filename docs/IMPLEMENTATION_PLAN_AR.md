# خطة التنفيذ التفصيلية - نظام First Click ERP

## نظرة عامة

نظام ERP محاسبي متعدد المستأجرين (Multi-Tenant) يعمل بنظام اشتراكات سنوية، مبني بـ:
- **Backend:** Laravel 12
- **Frontend:** React 18 + TypeScript + Vite
- **Database:** PostgreSQL
- **Hosting:** Cloud (AWS / Azure / Google Cloud)

---

## المرحلة 1: البنية التحتية وقاعدة البيانات العامة

### 1.1 إنشاء قاعدة البيانات العامة
- جدول `tenants` (العملاء/الشركات)
- جدول `subscription_plans` (خطط الاشتراك)
- جدول `subscriptions` (اشتراكات العملاء)
- جدول `tenant_users` (ربط المستخدمين بالعملاء وصلاحياتهم)
- جدول `users` (المستخدمين مع دعم 2FA)

### 1.2 بيانات العملاء الكاملة
| الحقل | النوع | الوصف |
|------|------|-------|
| name | string | اسم العميل/الشركة |
| address | text | العنوان |
| activity | string | النشاط (تجاري/صناعي/خدمي) |
| tax_registration_number | string | الرقم الضريبي |
| contacts | json | جهات الاتصال |
| email | string | البريد الإلكتروني |
| phone | string | رقم الهاتف |
| logo | string | شعار الشركة |

### 1.3 نظام الاشتراكات
- **أساسي:** ميزات محدودة، عدد مستخدمين أقل
- **متوسط:** ميزات إضافية، مستخدمين أكثر
- **متقدم:** كل الميزات، مستخدمين غير محدودين
- الترقية، التجديد التلقائي، التذكير قبل الانتهاء

## المرحلة 2: نظام Multi-Tenant

### 2.1 استراتيجية العزل
- **Row-Level:** `tenant_id` في كل جدول (مناسب للبداية)
- **Schema-per-Tenant:** مخطط منفصل لكل عميل (للشركات الكبيرة)
- **Database-per-Tenant:** قاعدة بيانات منفصلة (للأمان الأعلى)

### 2.2 تحديد المستأجر
- عبر Subdomain: `company1.firstclick.com`
- عبر Header: `X-Tenant-ID`
- عبر Token: معرف المستأجر في JWT

### 2.3 Middleware
- `EnsureTenantContext` - التأكد من وجود سياق المستأجر
- `CheckTenantSubscription` - التحقق من صلاحية الاشتراك
- `TenantIdentification` - تحديد المستأجر من الطلب

## المرحلة 3: دليل الحسابات الشجري

### 3.1 هيكل الحسابات
```
1 - الأصول
  1.1 - أصول متداولة
    1.1.1 - النقدية
    1.1.2 - المدينون
    1.1.3 - المخزون
  1.2 - أصول ثابتة
2 - الخصوم
  2.1 - خصوم متداولة
  2.2 - خصوم طويلة الأجل
3 - حقوق الملكية
4 - الإيرادات
5 - المصروفات
```

### 3.2 قابلية التعديل
- إضافة/تعديل/حذف حسابات فرعية
- ربط الحسابات بنشاط العميل (تجاري/صناعي/خدمي)
- قوالب جاهزة لكل قطاع

## المرحلة 4: القيود اليومية والحركات المالية

### 4.1 القيود التلقائية
- فواتير المبيعات → مدين: العملاء، دائن: المبيعات
- فواتير المشتريات → مدين: المشتريات، دائن: الموردين
- المصروفات → مدين: المصروفات، دائن: النقدية
- الإيرادات → مدين: النقدية، دائن: الإيرادات

### 4.2 مراكز التكلفة
- ربط كل حركة بمركز تكلفة اختياري
- تقارير حسب مراكز التكلفة

### 4.3 العملات المتعددة
- جدول `currencies` و `exchange_rates`
- تحويل تلقائي عند الإدخال
- تقارير بعملة أساسية

## المرحلة 6: المخزون

- كود الصنف، الوصف، الوحدة، التصنيف
