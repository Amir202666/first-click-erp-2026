# دليل دائم: المحلي (أوفلاين) → الأونلاين — بدون نقص

## القاعدة الذهبية

| ماذا تغيّر محلياً | ماذا ينقله `git push` | ماذا تحتاج إضافياً |
|-------------------|----------------------|---------------------|
| شاشات، إصلاحات، كود | نعم | `deploy` على السيرفر |
| عملات، فروع، مراكز تكلفة | ملف JSON في Git | `import` على السيرفر |
| عملاء، فواتير، مخزون كامل | لا (قاعدة بيانات) | نسخة MySQL كاملة (بحذر) |

**لا يُرفع أبداً:** `backend/.env` — بيانات السيرفر تبقى على السيرفر.

---

## سير العمل اليومي (موصى به)

### على جهازك — كل مرة

```bat
cd /d "d:\erp projects\first click"

REM 1) شغّل واختبر محلياً
scripts\local-dev.cmd

REM 2) عند الرضا — انشر (قائمة تفاعلية)
scripts\publish-to-online.bat
```

اختر:
- **1** — كود فقط  
- **2** — عملات/فروع/مراكز تكلفة فقط  
- **3** — كود + بيانات مرجعية (الأكثر شيوعاً)  
- **4** — قاعدة كاملة (خطر — يستبدل الإنتاج)

### على السيرفر — أمر واحد دائماً

```bash
bash /var/www/erp/deploy/publish-online.sh
```

يفعل تلقائياً:
1. `git pull`  
2. `deploy.sh` (بناء، migrations، nginx)  
3. استيراد `scripts/sync-data/reference_first-company.json` إن وُجد  
4. يعرض رقم النسخة `/deploy-revision.txt`

### في المتصفح — بعد كل نشر

- **Ctrl+Shift+R** مرة واحدة (أو نافذة خاصة)  
- تحقق: https://firstclickerp.top/deploy-revision.txt = آخر commit على GitHub

## قائمة تحقق (لا تنقص)

### قبل الرفع من المحلي

- [ ] اختبرت على `http://localhost:5173`
- [ ] `php artisan migrate` نجح محلياً إن أضفت migration
- [ ] لم تُضف ملفات `.env` للـ commit

### بعد الرفع

- [ ] شغّلت `publish-online.sh` على السيرفر
- [ ] `/deploy-revision.txt` يطابق `git log -1` على GitHub
- [ ] `/api/health` يعرض `{"ok":true}`
- [ ] حدّثت المتصفح (Ctrl+Shift+R)

### إن أضفت عملات/فروع/مراكز تكلفة

- [ ] شغّت `scripts\publish-to-online.bat` واخترت **2** أو **3**
- [ ] أو يدوياً: `scripts\push-reference-to-github.bat`

## مشاكل شائعة — لن تتكرر إن اتبعت الدليل

| المشكلة | السبب | الحل |
|---------|--------|------|
| التعديلات لا تظهر | السيرفر لم يُحدَّث أو كاش قديم | `publish-online.sh` + Ctrl+Shift+R |
| الشريط بالاتجاه الخاطئ | JS قديم في المتصفح | Ctrl+Shift+R / نافذة خاصة |
| عملات/فروع ناقصة | البيانات لم تُزامَن | خيار **2** أو **3** في `publish-to-online.bat` |
| فشل SSH من Windows | طبيعي على Hostinger | استخدم Git للبيانات + Terminal للسيرفر |
| `tenant:sync-reference` غير معرّف | كود قديم على السيرفر | `git reset --hard origin/main` ثم `publish-online.sh` |

## أوامر سريعة

| المهمة | جهازك | السيرفر |
|--------|--------|---------|
| تطوير يومي | `scripts\local-dev.cmd` | — |
| نشر كل شيء | `scripts\publish-to-online.bat` | `bash deploy/publish-online.sh` |

| قاعدة كاملة | `scripts\export-local-db.bat` | `scripts/sync-database.sh` |
| تحقق | `scripts\verify-production.bat` | `curl .../deploy-revision.txt` |

## ترتيب العمل الصحيح (ملخص)

```
محلي: عدّل → اختبر → publish-to-online.bat
         ↓
GitHub: main محدّث (كود + optional sync-data JSON)
         ↓
سيرفر: bash deploy/publish-online.sh
         ↓
متصفح: Ctrl+Shift+R → تحقق
```

بهذا الشكل **كل تعديل** يمر بنفس المسار — بدون نقص في الكود ولا في البيانات المرجعية.
